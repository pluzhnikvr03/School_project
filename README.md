# School Library Bot

**School Library Bot** — это полноценный Telegram-бот для автоматизации работы школьной библиотеки, разработанный на Python. Проект решает проблему бумажного учёта учебников и позволяет полностью цифровизировать процессы выдачи и возврата книг. Бот предназначен для трёх категорий пользователей: учеников, учителей и администратора, каждая из которых имеет свой функционал и уровень доступа.

## Ключевой функционал

**Для учеников** реализован самый востребованный функционал: при сканировании QR-кода, наклеенного на форзац учебника, бот автоматически распознаёт код и показывает подробную информацию о книге — название, автора и год издания. После этого ученику предлагаются три действия: «Взять», «Вернуть» или «Кому принадлежит?». Кнопка «Кому принадлежит?» позволяет проверить статус книги. Для ученика она показывает, свободна ли книга или занята (без указания личности). Для учителя отображается полная информация о том, у кого именно находится учебник, включая ФИО и класс. Команда `/books` формирует список всех учебников, которые в данный момент находятся на руках у пользователя, с указанием даты выдачи. Это удобно для контроля и планирования.

**Для учителей** предусмотрен расширенный функционал. После регистрации и подтверждения администратором учитель получает доступ к режиму помощи (`/help`), который активируется специальной командой. В этом режиме учитель может выбрать класс, затем конкретного ученика из списка, после чего все его действия с книгами (взятие и возврат) будут применяться к выбранному ученику. Это особенно полезно в экстренных ситуациях: если ученик забыл телефон, потерял учебник или просто отсутствует, учитель может оперативно решить вопрос. Команда `/status` показывает, за кого сейчас действует учитель, а `/stop_help` возвращает в обычный режим.

**Для администратора** реализован самый полный контроль. Все заявки на регистрацию учителей приходят администратору в личные сообщения с удобными inline-кнопками «Подтвердить» и «Отклонить». При подтверждении учитель получает доступ к боту, при отклонении его данные полностью удаляются из базы, и он может попробовать зарегистрироваться снова.

## Техническая реализация

Проект написан на Python и использует современные библиотеки. **pyTelegramBotAPI** отвечает за всё взаимодействие с Telegram: обработку команд, inline-клавиатуры, callback-запросы и отправку сообщений. **qrcode** и **Pillow** работают в связке для генерации QR-кодов — первая создаёт структуру кода, вторая превращает её в изображение и сохраняет в формате PNG. Все сгенерированные QR-коды автоматически помещаются в папку `qrcodes` с именами, соответствующими кодам книг, что упрощает навигацию и повторную печать.

Для хранения данных используется **встроенная библиотека SQLite** (`sqlite3`), которая не требует установки отдельного сервера и идеально подходит для локальных проектов. В базе данных созданы три таблицы: `users` хранит информацию о пользователях (Telegram ID, ФИО, класс, статус ученик/учитель, разрешение), `books` содержит данные об учебниках (QR-код, название, автор, год издания), а `records` фиксирует все операции выдачи и возврата с датами. Благодаря такому подходу вся история действий сохраняется и может быть проанализирована при необходимости.

Библиотека **os** используется для работы с файловой системой: она проверяет наличие папки `qrcodes` и создаёт её при необходимости, а также корректно формирует пути к файлам независимо от операционной системы (Windows, Linux или macOS).

## Безопасность и надёжность

В проекте предусмотрены механизмы защиты от распространённых проблем. При одновременной регистрации нескольких пользователей функция `register_user` использует повторные попытки и случайные задержки, чтобы избежать конфликтов при записи в базу данных. Обработка старых callback-запросов предотвращает ошибки после перезапуска бота. Токен бота хранится в отдельном файле `config.py`, который не попадает в публичный репозиторий.

## Структура проекта

Проект организован модульно для удобства поддержки и масштабирования. **botcode.py** содержит всю логику работы бота: обработчики команд, inline-кнопки, состояния диалогов. **database.py** отвечает за взаимодействие с SQLite: создание таблиц, функции для регистрации, выдачи, возврата и получения информации. **qr_generator.py** генерирует QR-коды в разных форматах (строка, кортеж, словарь) для гибкой интеграции. **config.py** хранит токен бота. Файл **requirements.txt** содержит все необходимые зависимости для быстрой установки на новом компьютере.

## Настройка Google Drive API

1. Перейди в [Google Cloud Console](https://console.cloud.google.com/)
2. Создай проект, включи **Google Drive API**
3. Настрой **OAuth consent screen** (External, добавь свой email в тестовые пользователи)
4. Создай **OAuth Client ID** (тип: Desktop app)
5. Скачай JSON → переименуй в `credentials.json` → положи в папку с ботом

## Перспективы развития

Проект спроектирован с учётом возможного расширения. В будущем можно добавить панель статистики для завуча с графиками популярности книг и динамикой выдачи, импорт списка учебников из Excel или CSV для массового добавления, а также уведомления о просрочке возврата. Архитектура позволяет легко интегрировать новые функции без серьёзной перестройки существующего кода.

School Library Bot — это готовое решение для любой школы, которое не требует сложной настройки, работает на любом компьютере и делает процесс учёта учебников простым и прозрачным для всех участников.
